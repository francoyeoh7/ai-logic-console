## 第 2 页 · 我们解决什么问题

**一句话：传统游戏里 NPC 不像人，他们像自动售货机。**

- NPC 靠有限状态机和对话树运行——你选 3 个选项，他们回复预设台词
- NPC 不记得你昨天干了什么，同一句问候说一万次
- 任务系统是「走到 X，触发 Y」——没有惊喜，没有变化
- 如果想让 NPC 有性格和记忆，策划需要写海量分支文本——不现实

**我们做的：把 NPC 的行为决策权交给一套可控的 AI 系统**

## 第 3 页 · 产品定位

**这是一个游戏编辑器，不是聊天机器人。**

- 定位等价于 UE5 蓝图编辑器——是内容生产工具
- 运行时不在游戏内——编辑器输出 JSON 配置文件给 UE5 加载
- 游戏发布后不需要编辑器——一切配置已编译为数据
- 类比：UE5 里有 DataTable 编辑器，我们有 AI 语义控制台

**三个「不是」：** 不是 ChatGPT 套壳 | 不是云端服务 | 不是运行时 GM 面板

## 第 5 页 · 系统架构

**三部分协作，控制台不参与运行时。**

```
控制台（Web 编辑器）    UE5（游戏引擎）    Ollama（推理服务）
       │                      │                    │
       │──  JSON 配置 ───────→│                    │
       │                      │──  HTTP 请求 ─────→│
       │                      │←── JSON 响应 ──────│
```

- UE5 直连 Ollama——不经过控制台，省一次网络跳
- 控制台只在编辑时导出 JSON——随时可关
- 游戏发布后零外部依赖

## 第 7 页 · 核心设计理念：双重决策

**LLM 只负责说话，不负责做决定。**

```
硬逻辑层（引擎决定）              软表达层（LLM 负责）
├─ 这个 NPC 能不能做这件事？       └─ 把决定用 NPC 的风格说出来
├─ 这个 NPC 应不应该做这件事？
├─ 有没有违反护栏规则？
└─ 有没有超出世界观边界？
```

为什么？7B 小模型做精确判断不稳定。但它把「已经决定好了」用不同风格说出来——非常稳定。

## 第 8 页 · NPC 人格八维度

**一个 NPC 维性格      贪婪 80  耐心 20  攻击性 60  魅力 40  忠诚 70
├─ 角色立场      商人（卖方）+ 情报源（信息发布者）
├─ 关系网络      对玩家信任 72 / 对卫队长信任 80[老朋友] / 对游商信任 20[戒备]
├─ 对话风格      粗暴、贪财、不耐烦 ← 由性格值自动生成
├─ 行为倾向      交易倾向 = 贪婪×0.5 + 信任×0.3 + 恐惧×(-0.2)
├─ 四层记忆      20 条工作记忆 + 50 条核心记忆 + 归纳印象
└─ 关联任务      主线狼患(情报源)、支线私藏(发布者)、AI闲聊(情报源)
```

## 第 9 页 · 四层记忆架构

**真人不是什么都记住——关键事件留下，不重要的事自然后忘掉。**

```
所有事件
    │
    ▼ 重要性门控（打喷嚏不记，亮武器记）
工作记忆（≤20 条，7 天自然衰减）
    │
    ▼ 被反复强化或关键时刻
核心记忆（≤50 条，永不删除）
    │
    ▼ 相似事件自动合并
世界印象  "这个人爱亮武器"（置信度 0.85）
```

**五种记忆来源：** 玩家行为 / 其他 NPC 行为 / 目睹事件 / 环境变化 / 自身经历

## 第 10 页 · 五层安全护栏

**AI 输出的每一句话在被玩家看到前，先过五道关卡。**

| 层 | 防线 | 检测什么 |
|---|---|---|
| 1 | 世界观边界 | 35 个现代词汇黑名单——"手机""AI""飞机" 不能说 |
| 2 | 能力边界 | 老韩不能声称"我去帮你杀龙" |
| 3 | 叙事连贯性 | 不能提到不存在的地名、不存在的 NPC |
| 4 | 知识真实性 | 不能凭空给任务、不能夸大奖励金额 |
| 5 | 紧急兜底 | 连续违规→沉默。超时→跳过。崩溃→缓存回退 |

**审核流水线：** LLM 输出 → 五层检查 → 不合格则注入矫正 Prompt 重新生成 → 二次不合格则回退兜底行为

## 第 11 页 · 任务系统的三层结构

**主线固定、支线半控、AI 随机——三层叠加不冲突。**

| 层 | 谁编排 | AI 做什么 |
|---|---|---|
| 主线任务 | 策划手工编节点链 | LLM 只负责每个节点里 NPC 的对话和情绪 |
| 支线任务 | 策划定义触发条件 + 节点链 | LLM 负责对话内容 |
| AI 动态任务 | 策划定边界（类型/数量/奖励范围） | 引擎按边界随机化，LLM 填充对话 |

**AI 任务不是随机刷——是从 NPC 记忆里生长出来的。**
老韩记得北境狼群多 → AI 任务方向锁定"狼" → 在边界内随机化「收集 3-5 张狼皮」

## 第 13 页 · 对话沙盒

**不启动 UE5 也能测试——改了参数立刻看效果。**

- 将 NPC 设定里的全部参数编译为一段「人格画像」System Prompt
- 聊天时自动注入对话历史（最近 8 轮），NPC 不会忘记之前说过什么
- 右侧实时显示：当前 NPC 的性格快照、关系列表、对话风格标签、关联任务
- 「重新装载」按钮——在 NPC 设定里改了参数，一键刷新，对话即用新设定

## 第 14 页 · 完整叙事案例

**6 个主线节点 × NPC 记忆 × 玩家选择 = 无限差异化的体验。**

| 节点 | 固定内容 | 变化来自 |
|---|---|---|
| 一 | 卫队长给任务，猎狼 | 老韩的好感决定他给不给额外线索 |
| 二 | 老韩带去矿洞 | 信任值决定他能否深入 vs 在洞口等 |
| 三 | 游商出现，买/留碎片 | 她的倾向公式决定说多少实话 |
| 四 | 军方封锁，三条路可选 | 不同 NPC 的关系值决定路线难度和对话 |
| 五 | 深入龙眠峰，面对远古存在 | 菲娜的家族记忆达到情绪峰值——LLM 读到 |
| 六 | 三种结局 | NPC 命运不由分支标签——由一路上的 trust/affection 综合决定 |

## 第 16 页 · 怎么接入 UE5

**控制台不是插件——它输出 JSON，游戏自己加载。**

控制台导出 3 个文件 →
  DT_NpcPersonas.json       = NPC 人设
  DT_QuestDefinitions.json  = 任务定义
  DT_GuardrailAxioms.json   = 护栏规则

UE5 程序员需要做的：
- 创建 3 个 C++ Struct（80 行代码，控制台已自动生成）
- 导入 JSON → DataTable（UI 拖拽操作）
- GameInstanceSubsystem 加载数据（70 行代码）

**总工作量：约 150 行 C++ + 3 次 UI 拖拽。一个程序员半天就能接好。**
自动生成《UE5 程序员对接文档》，包含完整代码可以直接复制。

## 第 18 页 · 总结

**我们做的不是一个聊天机器人。**

**我们做的是一套「AI 原生游戏开发工具链」：**
策划用控制台配置 NPC 的灵魂和世界的规则，
引擎在规则内运行，LLM 在边界内表达，
玩家经历的是策划预见了方向但没预写每一句话的故事。

**三个核心价值：**
1. 降低 AI NPC 设计门槛——策划会填表就能配置，不需要会写 Prompt
2. 保证行为可控——五层护栏 + 双重决策，LLM 不会乱来
3. 全部本地部署——零云端成本，独立游戏工作室同样用得起
