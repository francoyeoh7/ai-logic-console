## 1. 产品概述

### 1.1 这是什么

AI 语义与逻辑控制台是一个**独立于游戏引擎的 AI 内容编排工具**。它的定位等价于 UE5 蓝图编辑器或 Unity Prefab 编辑器——专门用来配置由大语言模型驱动的 NPC 行为系统。

### 1.2 它不是

- ✗ 聊天机器人或 ChatGPT 套壳
- ✗ 运行时 GM 调试面板
- ✗ 游戏本体的一部分
- ✗ 云端服务（100% 本地推理）

### 1.3 它是

- ✓ 游戏编辑阶段的**内容生产工具**
- ✓ NPC 人格、任务、记忆、护栏的**配置编辑器**
- ✓ 与传统 DataTable 等价的**结构化数据源**
- ✓ 支持导出 JSON → UE5 DataTable 的**离线工具**

### 1.4 解决的核心问题

传统游戏开发中 NPC 行为依赖三种手段：

| 手段 | 问题 |
|---|---|
| 有限状态机 | 只能预定义状态跳转，缺乏人格一致性 |
| 对话树 | 3-5 个固定选项，玩家被限制在策划预设的轨道上 |
| 行为树 | 适合战斗/移动，不适合社交和对话 |

AI 开放世界的核心需求：**同一个 NPC、同一套性格、同一段记忆——面对不同玩家行为产生自然、连贯、有差异但不失控的回应。** 这是传统工具做不到的。

### 1.5 技术参数

| 项目 | 规格 |
|---|---|
| 前端框架 | React 18 + TypeScript + Vite |
| 状态管理 | Zustand（单一 Store，跨模块共享） |
| UI 框架 | Tailwind CSS + Radix UI + Lucide Icons |
| 可视化 | React Flow（任务网络图） + Recharts（导演图表） |
| 本地推理 | Ollama（qwen2.5:7b, 4bit 量化, 4.7GB VRAM） |
| 硬件要求 | NVIDIA RTX 3060 12G 或更高 |
| 推理延迟 | 首轮 2-5 秒，后续 1-3 秒 |
| NPC 容量 | 同场景 5-20 个活跃 NPC |

## 2. 系统架构

### 2.1 三进程拓扑

```
┌──────────────────────┐         ┌──────────────────────┐         ┌──────────────────────┐
│   控制台（浏览器）     │         │   UE5 游戏引擎         │         │   Ollama 推理服务     │
│                      │  JSON   │                      │  HTTP   │                      │
│   React + TypeScript │────────→│   GameInstance       │────────→│   qwen2.5:7b          │
│   Zustand Store      │  导出   │   Subsystem          │ :11434  │   本地 GPU 推理       │
│                      │←────────│                      │←────────│                      │
│   7 个编辑面板        │WebSocket│   行为调度 + 状态采集  │  响应   │   结构化 JSON 输出    │
│                      │ (调试)  │                      │         │                      │
└──────────────────────┘         └──────────────────────┘         └──────────────────────┘
```

**关键设计决策：**
- UE5 **直连** Ollama，不经过控制台中转——省一次网络跳
- 控制台只在编辑时导出 JSON。运行时调试可选 WebSocket，但不是必须
- 控制台可以随时重启，不影响游戏运行

### 2.2 NPC 一次决策的全链路

```
T+0ms     UE5 检测到事件（玩家进店、身上有血）
T+1ms     上下文编排器打包 4 段 System Prompt:
            [A:人格画像] [B:最近记忆] [C:当前感知] [D:可用行为]
T+2ms     HTTP POST → Ollama（结构化 JSON 输出）
T+600ms   收到 JSON: {"action":"SPEAK","params":{"message":"剑收起来"}}
T+601ms   穿五层安全防线: 白名单 ✓ → 认知边界 ✓ → 公理引擎 ✓
T+602ms   UE5 执行: 播放说话动画 + 显示字幕
T+603ms   写入记忆 + 更新关系值（信任-3, 敬畏+5）
```

## 4. 模块详解

### 4.1 护栏与公理

**功能：** 设定不可被 LLM 幻觉绕过的物理与剧情铁律。

#### 4.1.1 五层防线

| 层级 | 防线 | 检测方式 | 违规示例 | 修正策略 |
|---|---|---|---|---|
| 1 | 世界观边界 | 35 个现代词汇黑名单 | NPC 说"我会用手机联系你" | 注入矫正 Prompt 重生成 |
| 2 | 角色能力边界 | 行为白名单 + 能力标签 | 老韩说"我去帮你杀那头龙" | 截断 → 矫正 → 重调 |
| 3 | 叙事连贯性 | 地名白名单 + 事实锚定 | 提到不存在的地点或 NPC | 地名/人名校验 |
| 4 | 知识真实性 | 任务锚定 + 奖励锚定 | 凭空给任务或夸大奖励 | 对比 QuestDefinition 校验 |
| 5 | 紧急兜底 | 连续两次违规 / 超时 15s | — | 回退到预设安全行为 |

#### 4.1.2 公理构建器

IF-THEN 级联规则，不可绕过：

```
玩家等级 < 10 AND 请求购买神器 → 强制覆盖为 FORCE_REJECT
NPC 敌意 > 0.5 AND 玩家行为 = 谈判 → 强制覆盖为 FORCE_ATTACK
玩家持有皇家印章 AND NPC 是守卫 → 强制覆盖为 FORCE_RESPECT
```

#### 4.1.3 预输出审核流水线

LLM 生成的文本**不直接展示给玩家**——先经过审核引擎：

```
LLM 输出 → auditResponse() 检查五层防线
    ├─ 全部通过 → 展示给玩家
    └─ 未通过 → generateCorrectionPrompt() 生成矫正 Prompt
               → 重新调用 LLM
               → 二次检查
               ├─ 通过 → 展示
               └─ 未通过 → 回退兜底行为
```

#### 4.1.4 兜底策略

| 情景 | 行为 |
|---|---|
| LLM 连续两次违规 | DO_NOTHING — 沉默 |
| LLM 超时 >15s | 播放若有所思动画 + 跳过本轮 |
| JSON 格式错误 | 正则部分解析 + fallback "……嗯。" |
| Ollama 服务崩溃 | 缓存最后一次合法 JSON，循环使用 + 通知"NPC 在发呆" |

### 4.2 NPC 设定

**功能：** 定义 NPC 完整人格画像——8 个维度构成。

#### 4.2.1 人格构成

```
NPC: 酒馆老板·老韩
├── 核心人设 (BasePrompt)
│   └── "你是一个脾气暴躁的老头，在乱世中经营酒馆..."
├── 5 维性格 (Traits)
│   ├── 贪婪 80  耐心 20  攻击性 60  魅力 40  忠诚 70
│   └── 每个维度 0-100，驱动对话风格和决策权重
├── 角色标签 (RoleTags)
│   ├── 商人 → 卖方立场：报价不降价
│   └── 情报源 → 掌握信息，有发布权
├── 关系网络 (Relationships)
│   ├── 对玩家: 信任 72 / 好感 70 / 敬畏 30
│   ├── 对卫队长: 信任 80 / 好感 70 / 敬畏 25 [老朋友]
│   └── 对游商: 信任 20 / 好感 30 / 敬畏 10 [戒备]
├── 四层记忆 (Memories + Impressions)
│   ├── 工作记忆 ≤20 条，7 天衰减
│   ├── 核心记忆 ≤50 条，永久保留
│   ├── 5 类来源: 玩家行为 / NPC 行为 / 目睹 / 环境 / 自身
│   └── 世界印象: 归纳认知而非具体事件
├── 对话风格 (DialogueStyles)
│   ├── 粗暴 → "用短句，语气粗鲁"
│   ├── 贪财 → "卖方立场，抬高报价，绝不降价"
│   └── 不耐烦 → "讨厌被追问同一个问题"
├── 意图映射 (IntentMappings)
│   ├── 交易: 0.7  抱怨: 0.5  打听: 0.4
│   └── 每种意图的权重决定 NPC「会聊什么」
└── 倾向公式 (InclinationFormulas)
    ├── 交易决策 = 贪婪×0.5 + 信任×0.3 + 恐惧×(-0.2)
    └── 信息透露 = 忠诚×(-0.4) + 好感×0.5 + 信任×0.3
```

#### 4.2.2 四层记忆架构

```
所有事件
    │
    ▼ 重要性门控 (imp ≥ 3)
工作记忆 (≤20 条)
    │ 7 天衰减
    │ 被印象引用 → 升级
    ▼
核心记忆 (≤50 条)
    │ Pinned 或 imp ≥ 8，永不删除
    │ 相似事件合并
    ▼
世界印象 (≤30 条)
    "这个客人爱亮武器" (置信度 0.85)
    "艾德是个可靠的老朋友" (置信度 0.90)
```

**记忆来源分类：** 玩家行为 / NPC 行为 / 目睹 / 环境 / 自身。每条记忆有来源标签、关联实体、发生地点、重要度、衰减速率、派生效果（关系影响 + 意图触发 + 任务标记）。

#### 4.2.3 关系网络与情绪传播

NPC 之间的关系不是广播——只在事件触发时传播：

```
玩家在酒馆骂了老韩
    │ 老韩 anger +30
    ├── 传播半径 15m:
    │   ├── 卫队长 (距离 8m, 朋友, 放大系数 1.5)
    │   │   ╰─ 愤怒 +27 → 对玩家的 trust -5
    │   └── 游商 (距离 5m, 陌生人, 系数 1.0)
    │       ╰─ 愤怒 +18
    └── 半径外: 炼金士 (距离 500m) → 不受影响
```

#### 4.2.4 对话风格自动生成

根据 5 维性格值自动映射风格标签：

| 性格 | 阈值 | 标签 | Prompt 注入 |
|---|---|---|---|
| aggression > 60 | >60 | 粗暴 | "用短句，语气粗鲁。但你是服务者——对顾客粗鲁但不敌视。" |
| patience < 30 | <30 | 不耐烦 | "讨厌被追问。但你是服务提供者，不是急着走的路人。" |
| greed > 65 | >65 | 贪财 | "卖方立场。抬高报价，绝不轻易降价。不是买方。" |
| loyalty > 65 | >65 | 忠诚 | "经常提到自己的阵营，为阵营辩护。" |

#### 4.2.5 倾向公式引擎

LLM 不负责决策——只负责表达。硬决策由公式做：

```
交易决策: 贪婪×0.5 + 信任×0.3 + 恐惧×(-0.2)
  ↓ 得分=51 > 阈值 40 → "决定交易"
  ↓ LLM 收到: "你决定接受这笔交易，但价格还可以再谈"
  ↓ LLM: "行吧，不过 1000 太少了，1500 怎么样"
```

### 4.3 任务网络图

**功能：** 以玩家为中心的无限画布，展示所有任务及其关系。

基于 React Flow 渲染，6 种自定义节点：

| 节点类型 | 颜色 | 内容 |
|---|---|---|
| PlayerCenter | 蓝色 | 玩家图标，固定在画布中央偏左 |
| MainQuest | 红色 | 主线任务 · 名称 + 等级区间 + 前置条件 |
| SideQuest | 绿色 | 支线任务 · 名称 + 触发条件 |
| AiQuest | 白色/灰色 | AI 动态模板 · 触发频率 |
| NpcNode | 黄色 | NPC 节点 · 角色标签 |
| RewardNode | 紫色 | 奖励节点 · 金币/物品/声望 |

**连线含义：**
- 实线 = 任务推进（A 完成 → B 解锁）
- 虚线 = AI 动态关联（可能触发）
- 双线 = 必要条件（A 必须完成才能做 B）

**布局规则：** 主线左上垂直排列分章（章节推进），支线左下自由分布，AI 模板右上网格排列，NPC 节点散布四周。

点击任务节点 → 底部弹出详情 + 「进入编排」按钮。

### 4.4 任务编排

**功能：** 配置任务的具体结构——主线节点链、AI 支线边界、奖励池。

#### 4.4.1 任务分类

| 类型 | 编排方式 | LLM 角色 |
|---|---|---|
| **主线任务** | 策划手工编排节点链（对话→收集→战斗→交任务），每节点有完成条件和下一个节点 | LLM 只负责每个节点的对话和情绪反应 |
| **支线任务** | 策划定义触发条件 + 节点链 | LLM 负责对话，触发条件由引擎判断 |
| **AI 动态任务** | 策划定义边界（任务类型/目标数量范围/奖励范围/稀有概率） | LLM 填充对话内容，引擎在边界内随机化 |

#### 4.4.2 AI 动态任务边界配置

| 配置项 | 内容 |
|---|---|
| 触发条件 | NPC 好感度 ≥ X、玩家等级 ≥ X、特定物品持有、特定地区 |
| 触发频率 | 每 N-M 次访问随机触发，概率可调 |
| 任务边界 | 任务类别（收集/战斗/递送/探索）/ 目标数量范围 / 目标区域 |
| 奖励边界 | 金币范围 / 经验范围 / 声望范围和阵营 |
| 稀有奖励 | 触发概率 % / 稀有物品池（每个物品有权重） |

### 4.5 导演规则

**功能：** 配置全局心流监测参数和干预事件池。导演 AI **不手动注入事件**——这是编辑器，不是运行时 GM 面板。

#### 4.5.1 六维心流指标

| 指标 | 数据来源 | 理想区间 |
|---|---|---|
| 上次有意义交互距今 | 战斗/对话/拾取时间戳 | < 45s |
| 5min 战斗频次 | CombatComponent 触发计数 | 1-2 次 |
| 资源压力指数 | HP% + 弹药% + 背包空格 | 40-70% |
| 空间探索熵 | NavMesh 新区域百分比 | 持续有新区域 |
| 主线推进间隔 | 任务完成时间差 | < 15min |
| 输入节律 | 暂停/菜单/读档频率 | 连续输入 |

#### 4.5.2 梯度干预

| 等级 | 无聊累计 | 高压累计 | 响应 |
|---|---|---|---|
| LIGHT | 30-90s | 30-60s | 天气变化、NPC 自言自语提示 |
| MEDIUM | 90-180s | 60-120s | 游商出现、小遭遇战 |
| HEAVY | >180s | >120s | 任务 NPC 主动搭话、远处冒烟暗示事件 |

### 4.6 上下文编排器

**功能：** 控制每次 LLM 调用发给模型的 Prompt 内容，保证可控且不超预算。

#### 4.6.1 四段式 Prompt 拼装

| 段落 | Token 预算 | 内容 | 可编辑 |
|---|---|---|---|
| [A] 核心公理 | ≤400 | NPC 人格 System Prompt + 世界规则铁律 | 否（自动填充） |
| [B] 最近记忆 | ≤600 | 语义检索 Top 6 记忆（含 Pinned） | 否 |
| [C] 当前感知 | ≤400 | 视线内实体 + 距离 + 状态 + 最近对话 | **是** |
| [D] 可用行为 | ≤400 | 当前状态下允许执行的行为 + 参数说明 | 否 |
| **总计** | **1800** | | |

包含 Token 预算实时计数器和行为白名单预览。

### 4.7 对话沙盒

**功能：** 不启动 UE5 也能测试 NPC 对话表现——核心测试工具。

#### 4.7.1 人格画像编译器

不是一个简单的聊天机器人——每次对话前，系统将 NPC 在 NPC 设定中配好的全部内容编译为完整的人格画像 System Prompt，包括：

- 核心人设（BasePrompt）
- 5 维性格数值
- 角色定位 + 社交立场声明（merchant = 卖方 vs informant = 信息发布者）
- 关系网络（对每个实体的 trust/affection/fear）
- 对话风格标签 + Prompt 注入指令
- 行为倾向公式
- 当前情绪状态
- 关联任务背景

#### 4.7.2 对话上下文注入

不是每一轮只发「玩家刚刚说: X」——系统将**完整对话历史**（最近 8 轮）注入 Prompt，保证 LLM 能串起上下文。你问「多少钱」时，它知道你在问上一句提到的「情报」。

#### 4.7.3 重新装载按钮

在 NPC 设定页改了参数后，点击「重新装载」→ 立即重新编译人格画像 → 继续对话使用新设定。不需要离开对话沙盒。

## 5. 核心设计理念

### 5.1 双重决策模型

**原则：LLM 只负责表达，不负责决策。**

```
硬逻辑层（编辑器配置，引擎执行）        软表达层（LLM 负责）
├─ 行为白名单                          └─ 把「决定交易了」变成
├─ 倾向公式                                "行吧，不过 1000 太少了，
├─ 护栏公理                                1500 怎么样"
├─ 前置条件
└─ 世界观边界
```

为什么这样设计？7B 小模型做「该不该交易」这样的精确判断不稳定。但让它把「已经决定交易了」说出来——它可以做到风格多变、不重复、有人味。

### 5.2 四层记忆架构

借鉴认知心理学——人类记忆的核心是「遗忘」而非「存储」。

- 不重要事件（打碎杯子, imp=2）→ 几小时到几天自然衰减消失
- 重要事件（展示龙鳞碎片, imp=9）→ 自动升级为核心记忆，永久保留
- 相似事件（三次迟到）→ 合并为印象 "这个人没时间观念"，不保留三个原始事件
- 每 NPC 工作记忆 ≤20 条，核心记忆 ≤50 条，印象 ≤30 条——容量可控

### 5.3 关系网络与情绪传播

NPC 不只是数据和雕像——他们有对每个认识的人的态度。事件发生时，情绪通过关系网「传播」，形成连锁反应。

这比传统的「NPC 好感度条」更精细：信任、好感、敬畏是三个独立轴。一个 NPC 可能很信任你但不太喜欢你（功能性关系），也可能既信任又喜欢（亲切的熟人）。

### 5.4 五层护栏防线

不是简单的「加一个审核开关」。五层递进：

**认知边界 → 能力边界 → 叙事连贯性 → 知识真实性 → 紧急兜底**

每层有不同的检测机制（黑名单/正则/白名单/能力标签/锚定对比），违规后有矫正流程（注入矫正 Prompt → 重新生成 → 二次审核 → 兜底回退）。

### 5.5 倾向公式引擎

灵感来自 Chris Crawford 的状态变量理论。NPC 的行为不是 if-else 判断，而是**多个维度的加权求和**：

```
倾向值 = Σ(状态变量 × 权重)

交易倾向 = 贪婪×0.5 + 信任×0.3 + 恐惧×(-0.2) + 金额×0.1
→ 得分 > 阈值? → 决定行为方向 → LLM 表达
→ 得分 < 阈值? → 拒绝，不浪费 LLM token
```

同一个 NPC 同一种交易，今天接受、明天拒绝——因为恐惧值变了，不是策划写死的。

### 5.6 人格画像编译

对话沙盒不是独立聊天机器人——是 NPC 设定所有配置数据的「编译器」。将编辑器中分散在 8 个面板的参数编译为一段结构化 System Prompt，注入到 LLM。改了任何 NPC 参数后点「重新装载」立即生效。

## 6. 任务系统

### 6.1 主线任务

策划手工编排节点链。每个节点有：
- **触发条件**：对话 NPC、触发地点、前置任务
- **节点类型**：对话/战斗/收集/护卫/探索/递送
- **完成条件**：表达式（dialogue:accept / combat:kill / collect:完成要求的物品）
- **下一节点**：链式推进
- **奖励配置**：金币/经验/声望/物品

### 6.2 支线任务

手工编排，有触发条件但不强制推进主线。不同 NPC 给出不同支线，支线的可获得性受 NPC 好感度和玩家等级影响。

### 6.3 AI 动态任务

从 NPC 的记忆里「生长」出来——不是随机刷的。

生成流程：
1. 导演 AI 检测玩家无聊累计超阈值
2. 筛选当前区域内的 NPC
3. 读取 NPC 的核心记忆 → 找到与当前进度相关的记忆
4. AI 任务边界约束（任务类型、目标数量、奖励范围）
5. LLM 填充对话内容

### 6.4 叙事整合案例：龙眠之夏

这是一个已经设计好的 6 节点完整主线示例——策划配好节点链，运行时 NPC 的记忆和关系让每个玩家的体验不同：

| 节点 | 固定内容 | 变化来源 |
|---|---|---|
| 一：北境狼患 | 卫队长给任务，去北境猎狼 | 老韩对玩家的态度影响他给不给额外线索 |
| 二：地下的回音 | 老韩带去矿洞，发现龙息石 | 老韩对玩家的 trust 决定他能否深入 vs 在洞口等 |
| 三：菲娜的赌注 | 菲娜出现，买/留龙鳞碎片 | 菲娜的倾向公式决定她对玩家说多少实话 |
| 四：命运与骰子 | 军方封锁峡谷，三条路可选 | 不同 NPC 的关系值决定难度和路上的对话 |
| 五：龙息的余烬 | 深入龙眠峰，面对远古存在 | 菲娜的家族使命达到情绪峰值——LLM 读到这段记忆 |
| 六：火与契约 | 最终结局，三种分歧 | NPC 的命运不由「分支标签」决定——由一路上的 trust/affection/记忆综合计算 |

### 7.1 对接方案

控制台导出 3 个 JSON 文件 → UE5 导入 DataTable → GameInstance 启动时加载。

```
  │                         │
  ├─ DT_NpcPersonas.json ──→ DataTable
  ├─ DT_QuestDefinitions.json ──→ DataTable
  └─ DT_GuardrailAxioms.json ──→ DataTable
                                   │
                            AIConfigSubsystem
                           (GameInstanceSubsystem)
                                   │
                     ┌─────────────┼─────────────┐
                     ▼             ▼             ▼
                GetNpcConfig  GetQuestConfig  GetEnabledGuardrails
```

**UE5 C++ 代码量：约 150 行**

- `AIConfigTypes.h`: 3 个 C++ Struct（约 80 行）

## 8. 创新价值

### 8.1 与现有方案的对比

| 维度 | 传统对话树 | NVIDIA ACE | Inworld.ai | 本系统 |
|---|---|---|---|---|
| **决策权分配** | 策划 100% | LLM 主导 | LLM 主导 | **硬逻辑 + LLM 表达** |
| **NPC 记忆** | 无 | 短期上下文 | 云端存储 | **四层本地记忆** |
| **任务系统** | 完全固定 | 需额外开发 | 需额外开发 | **内置三层任务** |
| **离线能力** | N/A | 云端依赖 | 云端依赖 | **100% 本地** |
| **护栏机制** | 无 | 基础过滤 | 基础 | **五层防线 + 预输出审核** |
| **部署成本** | 一次性 | 按 token | 按月付费 | **一次性部署** |

### 8.2 革新的玩法可能性

- **全自由对话解谜**：NPC 掌握关键线索但必须问对问题。可能撒谎（loyalty < 30），需要交叉验证
- **关系网的自动涌现**：NPC 之间的记忆互相传播，形成策划未写的连锁事件
- **伪装与渗透**：换装可以骗眼睛但骗不过记忆——以前偷过的东西不会消失
- **导演驱动的动态叙事**：不通过刷更强的怪来调节难度，而是通过更复杂的人际冲突和更深的剧情线索
