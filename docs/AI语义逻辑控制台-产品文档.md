## 1. 产品概述

### 1.1 定位

AI 语义与逻辑控制台是一个独立于传统 3D 地图编辑器的「AI 运行时中枢」。负责管理大语言模型驱动的 NPC 人格、安全护栏、行为输出、事件分发和全局导演 AI。

### 1.2 核心价值

| 问题 | 传统做法 | 本系统做法 |
|---|---|---|
| NPC 对话 | 预设对话树，3-5 个选项 | 自由文本输入，LLM 基于人格+记忆+感知做情境回应 |
| NPC 行为 | 有限状态机，固定动画循环 | LLM 输出结构化动作指令，经白名单校验后由引擎执行 |
| NPC 记忆 | 不记得玩家 | 语义向量检索 + 时间衰减 + 长期印象 |
| 游戏节奏 | 策划手工埋点 | 6 指标心流监测 + 3 级梯度干预 |
| 护栏 | 硬编码 if-else | 三道防线：输入过滤 → Schema 白名单 → 公理引擎 |

### 1.3 设计原则

- **渐进式披露** — 界面极简，逐步展开细节
- **暗色开发者美学** — 保持信息密度但避免视觉噪音
- **引擎解耦** — 控制台和游戏引擎通过 WebSocket + JSON 通信，互不依赖
- **本地优先** — 优先支持 Ollama/llama.cpp 本地推理，云端 API 为可选

## 2. 系统架构

### 2.1 三进程拓扑

```
┌─────────────────────┐          ┌─────────────────────┐          ┌─────────────────────┐
│  AI 语义控制台        │ WebSocket│  UE5 游戏端          │ HTTP     │  Ollama / llama.cpp   │
│  (React + TypeScript) │◄────────►│  (C++ Subsystem)     │◄────────►│  (qwen2.5:7b)         │
│                      │ :9090    │                      │ :11434   │                       │
│  配置编辑              │          │  行为调度 + 状态采集  │          │  推理 + 向量化         │
│  状态监控              │          │  导演 AI 运行时       │          │                       │
│  对话沙盒              │          │  记忆存储             │          │                       │
└─────────────────────┘          └─────────────────────┘          └─────────────────────┘
```

**连接关系：**

- **UE5 ↔ Ollama**: UE5 直连推理服务，控制台不代理（避免额外一跳延迟）。控制台只在对话沙盒中直接调 Ollama。
- **控制台 ↔ UE5**: 单条 WebSocket 双向通信。配置同步 + 状态监控 + 调试指令共用同一连接。
- **控制台 ↔ Ollama**: 仅对话沙盒和模拟调用场景。

### 2.2 单 NPC 一次决策全链路

```
T+0ms     UE5 检测到 NPC 感知范围内有新事件
T+1ms     上下文编排器拼装 4 段系统 Prompt
T+2ms     POST /v1/chat/completions
T+200ms   (8B 模型推理中)
T+600ms   收到结构化 JSON 响应
T+601ms   安全校验: 动作白名单 + 参数边界 + 护栏规则
T+602ms   行为入队 → ActionDispatcher
T+603ms   执行: 播放动画 + 显示字幕 + 广播事件
```

**关键约束:** 同一 NPC 两次 LLM 调用最小间隔 1.5s。间隔期内新事件缓存，下次调用时处理。

### 2.3 通信协议

WebSocket 统一信封：

```json
{
  "type": "message_type",
  "seq": 1,
  "timestamp": 1719676800000,
  "payload": { }
}
```

消息类型清单：

| 方向 | type | 频率 | 含义 |
|---|---|---|---|
| ↑ | `npc_status_snapshot` | 每 0.5s | 所有活跃 NPC 状态 |
| ↑ | `director_metrics` | 每 0.5s | 6 项心流指标 |
| ↑ | `llm_call_log` | 事件驱动 | LLM 调用摘要 |
| ↑ | `guardrail_fire` | 事件驱动 | 护栏触发记录 |
| ↓ | `config_sync` | 编辑时 | 增量/全量配置 |
| ↓ | `simulate_action` | 调试 | 手动注入行为指令 |

## 4. 模块 A: 灵魂锻造台

**用途：** 定义 NPC 底层人格、性格数值，管理高维向量记忆。

### 4.1 NPC 名册 (左侧)

- 可搜索、过滤的 NPC 列表
- 选中 NPC 后右侧面板联动

### 4.2 基础人格编辑器 (中间)

| 字段 | 类型 | 说明 |
|---|---|---|
| Core Prompt | Textarea (Markdown) | NPC 系统级指令，永不改变，不可被对话覆盖 |
| 贪婪度 | Slider 0-100 | NPC 对金钱和物品的追求程度 |
| 耐心 | Slider 0-100 | NPC 等待玩家或忍受冒犯的能力 |
| 攻击性 | Slider 0-100 | NPC 倾向于解决问题的程度 |
| 魅力 | Slider 0-100 | NPC 社交影响力 |
| 忠诚度 | Slider 0-100 | NPC 对阵营/雇主的忠诚程度 |

### 4.3 向量记忆修剪器 (右侧)

| 功能 | 说明 |
|---|---|
| 语义搜索 | 输入关键词检索相关记忆（需要 embedding 模型支持） |
| 记忆列表 | 展示时间戳、事件摘要、重要度评分 (1-10) |
| [Pin] | 固化为核心记忆，永远进入 LLM 上下文窗口 |
| [Forget] | 从向量库删除此条记忆 |

**记忆生命周期：**
- **ACTIVE**: 48 小时内可被检索
- **DORMANT**: 超过 48 小时自动沉降，仅话题命中时唤醒
- 每个 NPC 最多保留 200 条记忆

## 5. 模块 B: 护栏与公理

**用途：** 设定不可被 LLM 幻觉绕过的物理与剧情铁律。

### 5.1 全局安全开关

| 开关 | 说明 |
|---|---|
| 出戏检测 (Anti-OOC) | 阻止 NPC 做出不符合角色设定的回应 |
| 防提示词注入 (Anti-Prompt Injection) | 过滤用户输入中隐藏的注入攻击 |
| 兜底策略 | 4 选 1: 原地发呆 / 呼叫警卫 / 离开场景 / 使用上一个合法输出 |

### 5.2 公理构建器

采用 IF-THEN 级联菜单：

```
[IF 条件] → [THEN 强制动作]
```

示例公理：
- `玩家等级 < 10 AND 请求购买神器 → FORCE_REJECT`
- `NPC.敌意 > 0.5 AND 玩家行为 == '谈判' → FORCE_ATTACK`
- `玩家持有('皇家印章') AND NPC.类型 == '守卫' → FORCE_RESPECT`

**三道防线：**
1. **输入过滤** — 敏感词 + 注入模式检测
2. **Schema 白名单** — 动作必须在注册表中
3. **公理引擎** — IF-THEN 规则覆盖 LLM 输出

## 6. 模块 C: 事件总线路由图

**用途：** 可视化 LLM 提取的原子事件如何分发给游戏各子系统。

### 功能

- React Flow 无限画布
- 事件源节点 (左侧，如 `NPC_INSULTED`)
- 子系统节点 (右侧，如 `Reputation System`)
- 拖拽连线绑定事件→子系统
- 点击连线编辑权重因子 (如 `-30` 好感度、`multiplier:0.5` 经济)

### 支持的游戏子系统

| 子系统 | 含义 |
|---|---|
| reputation | 声望系统 |
| affection | 好感度 |
| economy | 经济系统 |
| wanted_level | 通缉等级 |
| experience | 经验值 |
| world_state | 世界状态（实体标记/移除） |

## 7. 模块 D: 导演节奏控制台

**用途：** 全局监控玩家心流，防止长期无聊或极度高压。

### 7.1 心流监测

6 项硬指标融合为心流指数：

| 指标 | 数据来源 | 理想区间 |
|---|---|---|
| 上次有意义交互距今 | 战斗/对话/拾取时间戳 | < 45s |
| 5min 战斗频次 | CombatComponent 触发计数 | 1-2 次 |
| 资源压力指数 | HP% + 弹药% + 背包空格 | 40-70% |
| 空间探索熵 | NavMesh 新区域百分比 | 持续有新区域 |
| 主线推进间隔 | 任务完成时间差 | < 15min |
| 输入节律 | 暂停/菜单/读档频率 | 连续输入 |

### 7.2 梯度干预

| 等级 | 无聊累计 | 高压累计 | 响应 |
|---|---|---|---|
| LIGHT | 30-90s | 30-60s | 天气变化、NPC 自言自语 |
| MEDIUM | 90-180s | 60-120s | 游商出现、小遭遇战 |
| HEAVY | >180s | >120s | 任务 NPC 搭话、远处冒烟 |

### 7.3 世界可能性池

导演不凭空生成事件——提前在 UE5 中埋「可能性对象」：

| 可能性类型 | 示例 | 前置条件 |
|---|---|---|
| 环境 | 区域可变的天气 | 室外 + 非地下 |
| NPC | 游商不定期出现 | 路边 + 白天 |
| 叙事 | 主线 NPC 主动找玩家 | 玩家 20min 未推进 |

## 8. 模块 E: 行为编排台

**用途：** 定义 NPC 所有可执行动作，与 UE5 C++ 类一一对应。

### 8.1 7 大行为类别

| 类别 | 动作示例 |
|---|---|
| 移动 (movement) | MOVE_TO, FLEE_FROM, APPROACH, PATROL_PATH |
| 对话 (speech) | SPEAK, SHOUT, WHISPER, MONOLOGUE, GREET |
| 社交 (social) | BOW, WAVE, KNEEL, HAND_OVER, REFUSE |
| 战斗 (combat) | ATTACK_MELEE, ATTACK_RANGED, DEFEND, SURRENDER |
| 交互 (interaction) | GIVE_ITEM, TAKE_ITEM, USE_OBJECT, OPEN_DOOR |
| 姿态 (posture) | SIT, STAND, SLEEP, WAKE, IDLE_POSE |
| 元动作 (meta) | CONTINUE_CURRENT, DO_NOTHING, OBSERVE |

### 8.2 单条行为记录结构

每条行为包含：
- **actionId**: 唯一标识
- **ue5Class**: 对应 UE5 C++ 类名
- **参数 Schema**: 名称/类型/是否必填/默认值/值域约束
- **前置条件**: 可执行的前提（视线内/未睡眠/可移动...）
- **执行效果**: 对其他子系统的影响
- **打断规则**: 可否打断/被更高优先级打断/互斥类别
- **动画引用**: UE5 AnimMontage 路径

## 9. 模块 F: 上下文编排器

**用途：** 控制每次 LLM 调用的上下文窗口内容，保证可控、安全。

### 9.1 四段式拼装

```
┌───────────────────────┬────────┬───────────────────────────┐
│ 段落                  │ Token  │ 内容                      │
├───────────────────────┼────────┼───────────────────────────┤
│ [A] 核心公理          │ ≤400   │ NPC 人格 System Prompt     │
│                       │        │ 世界规则铁律               │
├───────────────────────┼────────┼───────────────────────────┤
│ [B] 最近记忆          │ ≤600   │ 语义检索 Top 6 记忆        │
│                       │        │ 含 Pinned 固化记忆         │
├───────────────────────┼────────┼───────────────────────────┤
│ [C] 当前感知          │ ≤400   │ 视线内实体+距离+状态       │
│                       │        │ 最近 3 条对话/事件         │
├───────────────────────┼────────┼───────────────────────────┤
│ [D] 可用行为          │ ≤400   │ 当前状态下允许执行的行为    │
│                       │        │ 含参数说明                 │
├───────────────────────┼────────┼───────────────────────────┤
│ 总计                  │ 1800   │                           │
└───────────────────────┴────────┴───────────────────────────┘
```

### 9.2 功能

- 四段面板 (三个自动填充 + 一个可编辑 C 区)
- Token 预算实时进度条 (简易估算法: 英文≈0.25t/char, 中文≈0.75t/char)
- 右侧记忆检索预览 (当前 NPC 的 Top 6 记忆)
- 右侧行为白名单 (所有已注册动作)
- 「模拟调用」按钮 → 直接调 Ollama 看返回结果

## 10. 模块 G: 对话沙盒

**用途：** 不启动 UE5 也能测试 NPC 对话表现。

### 功能

- 左侧 NPC 选择列表 (来自灵魂锻造台的 NPC 名册)
- 中间聊天气泡界面 (玩家/ NPC 左右分布)
- 右侧响应检查器：显示 LLM 返回的 JSON 结构
  - 解析出的动作 + 对应行为定义
  - 参数字段
  - 情绪/优先级/推理原因
  - Token 消耗统计

### 使用流程

1. 选 NPC → 自动填充其 Core Prompt 到 Section A
2. 输入你想说的话
3. 系统拼装完整 Prompt → 调用 Ollama
4. 响应显示在聊天气泡中，原始 JSON 在右侧解析

## 11. 模块 H: 运行时监控

**用途：** 控制台作为游戏运行时仪表盘。

### 功能

- 活跃 NPC 状态网格 (实时刷新，展示 NPC ID/当前动作/情绪)

- 护栏触发日志 (规则 + 输入内容 + 拦截动作)
- WebSocket 连接状态指示器 + 心跳
- 手动注入指令 (调试用：直接向 UE5 发送行为指令)

## 12. 模块 I: 导出中心

**用途：** 将控制台配置导出为 UE5 可加载的格式。

### 导出格式

- **JSON 快照** (`.json`) — 完整配置，UE5 `config_sync` 加载

- **版本比对** — Diff 两个版本的配置变更

## 14. UE5 对接规格

### 14.1 需要的 C++ 模块 (约 2000 行)

| 类 | 职责 | 行数 |
|---|---|---|
| `UAIWorldCollector` | 采集 NPC 周围状态，序列化为文本 | ~300 |
| `UAIActionDispatcher` | 优先级队列 + 打断/过期 + 动画+字幕 | ~500 |
| `UAIDirectorRuntime` | 6 指标 + 心流融合 + 干预决策 | ~400 |
| `UAIMemoryStore` | SQLite 封装，向量存储+检索 | ~400 |
| `UWorldPossibilityPool` | 世界可能性注册+前置条件检查+权重随机 | ~350 |

### 14.2 推理服务要求

- 模型: Qwen2.5-7B (4bit 量化, 约 4.7GB VRAM)
- 推理速度目标: < 400ms/次
- 上下文窗口: 32768 tokens
- 硬件: NVIDIA RTX 3060 12G 或更高
